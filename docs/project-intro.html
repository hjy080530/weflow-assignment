04 · ARCHITECTURE

기술 구조

Next.js App Router 단일 풀스택. 외부 의존 없이 로컬에서 완결됩니다.

데이터 흐름

[폼 제출] ─POST→ /api/reservations · /api/inquiries
    ↓ lib/queries.ts (insert)
    ↓ SQLite (node:sqlite, data/weflow.sqlite)
    ↓ lib/realtime.ts (publishRealtimeUpdate)
    ↓ SSE /api/admin/stream
[관리자 대시보드] ←EventSource← useAdminTable() ─→ 즉시 갱신

레이어별 구성

레이어구현
프레임워크Next.js 16 App Router · React 19 · TypeScript strict (any 금지)
스타일Tailwind CSS 4 · Pretendard · 인라인 스타일 배제
데이터SQLite (Node.js 내장 node:sqlite, WAL 모드) — Supabase에서 전환
쿼리lib/queries.ts — prepared statement insert/get/update/delete
실시간lib/realtime.ts 인메모리 이벤트 + SSE 스트림
인증httpOnly 쿠키 세션 (admin-auth.ts), 미들웨어 보호
엑셀xlsx — 예약/문의/전체 통합 3종, 날짜 스탬프 파일명
테스트Playwright E2E 4종 + lint/tsc/build 검증

관리자 기능

실시간 반영

SSE 스트림 구독, 폼 제출 즉시 갱신, 15초 heartbeat

상태 관리

대기→진행중→완료 즉시 DB update + 옵티미스틱 UI

엑셀 다운로드

예약 / 문의 / 전체 통합(2시트) 3종

요약 통계

총건수·오늘접수·상태분포·전환율 카드 6종

코드 품질 규칙

  • TypeScript strict — any 미사용
  • Tailwind 반응형 (md:/lg:) 전면 적용
  • App Router only (pages/ 미사용)
  • 컴포넌트 default export 일관
  • data-testid 규칙 준수 (자동화 대응)
  • 폼 → DB insert → 관리자 반영 검증
  • 병원·의료 콘텐츠 전면 배제
  • ESLint + Husky + commitlint 게이트
